## Kit identity
A wide, slow, layer-and-hold sustain instrument. Seven drone families across 32 pads, organised by family then ascending pitch (the GM drum map is irrelevant for a sound-design kit). Uses 5/6 body models (Membrane, String, Plate, Shell, Bell), the three sustaining exciters (Friction, Feedback, Mallet+long-decay), the only kit-wide use of the sustained Filter ADSR, heavy DecaySkew tilt, modeStretch up to the new metallic B_max, deliberate 1/k ModeInject on bowed-membrane/ghost/temple pads, per-note Material Morph, Membrane-only Tension glide, head/shell coupling, west-coast Drive/Fold on the string drones, and equal-power Pan spreading all voices.

## Verification log (7 issues found & fixed)

1. COVERAGE (filter-env ADSR gap, friction pads 2,4,6 + membrane pads 3,5,7): the kit's stated signature is 'the only kit-wide use of the sustained Filter ADSR (long atk/dec/rel)', and Filter Env Amount is non-zero (0.55-0.62) so the filter env is ENGAGED and AUDIBLE on every friction pad. But only pads 0 and 1 set the full ADSR (Atk 0.2 / Dec 0.4 / Rel 0.65); pads 2,3,4,5,6,7 set ONLY Filter Env Sus 0.55 and silently inherit the struct defaults Atk 0.0 / Dec 0.1 / Rel 0.1 -> an INSTANT-attack, ~near-zero-decay env that contradicts the 'slow 83ms bloom / eases back / graceful fade' intent and the friction-string-drone recipe (slow 83ms atk / 333ms dec / 0.55 sus). These are meaningful-but-silently-defaulted params. FIX: carry Filter Env Atk 0.2, Filter Env Dec 0.4, Filter Env Rel 0.65 on all six friction/membrane pads (2,3,4,5,6,7), matching pads 0/1.

2. COVERAGE (filter-env ADSR gap, feedback pads 10,11,12,13): same gap on the Feedback half. The feedback-plate-shell-drone recipe specifies Filter Env Atk 0.45 (~46ms bloom), Dec 0.6 (~432ms), Rel 0.65 (~549ms) so 'the drone EVOLVES, no strike'. Pad 8 sets the full ADSR and pad 9 sets Atk 0.45 + Rel 0.65; but pads 10,11,12,13 set ONLY Filter Env Amount 0.65 + Sus 0.55 and inherit Atk 0.0 / Dec 0.1 / Rel 0.1, giving a near-instant filter attack on a self-oscillating metallic drone (a click/zap rather than a bloom). FIX: carry Filter Env Atk 0.45, Filter Env Dec 0.6, Filter Env Rel 0.65 on pads 10,11,12,13 (and add the missing Dec 0.6 to pad 9), matching pad 8.

3. PHYSICAL-RATIONALE ERROR (gong pads 28,29 Secondary Size rationale): the proposal states secondarySize 0.40 puts the 'shell above head for the metal ring' / 'shell sits above head'. Per param-dictionary offset 56, secondarySize maps sizeRatio = 1 - norm*0.75, so norm 0.40 -> shell f0 = 0.70*head_f0 -- BELOW the head, and the parameter can NEVER place the shell above the head (range is head..0.25*head). The value 0.40 is legal and fine for a sub-octave ringing-metal layer, but the stated reasoning is physically backwards. FIX: rationale corrected to 'shell f0 ~= 0.70*head_f0, a sub-octave ringing-metal layer (secondarySize cannot exceed head f0)'.

4. PHYSICS TENSION, kept-with-note (temple bells 30,31 + bowed-membrane Mode Inject): the indian-temple-bell archetype explicitly lists Mode Inject as a DELIBERATE DEFAULT-OFF because a 1/k integer harmonic series 'would add a contradictory integer series' on the inharmonic (Simpson-tuned hum/prime/tierce/quint = 0.25/0.5/0.6/0.75/1.0) bell -- the same reason singing-bowl and ghost-tone keep ModeInject 0. The proposal overrides this to 0.22 as a kit-character choice ('harmonic-alignment hum'). This is defensible for an Unnatural sound-design kit (the injected series sits at the body f0 = the nominal, reinforcing the perceived strike pitch, at only -modest amount 0.22) and is a documented deliberate delta, NOT an accidental wrong value, so I am KEEPING it but flagging the physical tension and tightening the rationale.
